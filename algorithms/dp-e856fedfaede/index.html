动态规划(一)—如何入门动态规划

基础

作为程序员来说，算法是必不可少的技能，不管是作为提升自己逻辑思维还是应付面试，而算法中的动态规划又是算法入门者的一个噩梦（当然对于笔者来说也是，毕竟笔者也不是天才）。

这一篇文章我将我自己学习动态规划的一些心得、体会分享出来，主要包括一些动态规划的基础，以及我学习动态规划的一些资料。

这一篇文章我们主要是对动态规划的一些概念做一些了解，如果你绝对这些概念晦涩难懂，不妨先看看后面的文章，再回过头来理解。

当然，我只是站在一个刚接触编程的大学生的视角来理解动态规划，如内容不正确，欢迎指正。

1. 引入

题目：数字三角形

在上面的例子中，可以很容易的得到最优路径是 7→3→8→7→57\rightarrow 3\rightarrow 8\rightarrow 7\rightarrow 57→3→8→7→5，总和是 303030。

那么最简单的思路就是从第一行的数开始，向下枚举所有的路径，但明显时间复杂度很高，可以达到 O(2r)O(2^r)O(2r)。

我们注意到，对于一条 最优的路径，它的每一步决策都是最优的。

以 7→3→8→7→57\rightarrow 3\rightarrow 8\rightarrow 7\rightarrow 57→3→8→7→5 为例，只考虑前四步 7→3→8→77\rightarrow 3\rightarrow 8\rightarrow 77→3→8→7，显然，这四步就是 r=4r=4r=4 时的最优解。

而对于每一个点，它的下一步决策只有两种，往左下或者右下（如果存在），因此只需要记录当前点的最大权值，用这个最大权值执行下一步决策，来更新后续点的最大权值。

另外一个好处在于，我们成功的缩小了问题的规模，将一个问题分成了多个规模更小的问题，也就是说，我们要想得到从顶端到第 rrr 行的最优方案，只需要知道从顶端到第 r−1r-1r−1 行的最优方案即可。

最后一个问题是，子问题间重叠的部分会有很多，同一个子问题可能会被重复访问多次，效率还是不高，那么解决的方法是将每个子问题的解存储下来，通过记忆化搜索的方式限制访问顺序，确保每个子问题只被访问一次。

以上就是 动态规划 的基本思路。

2. 什么是动态规划

2.1 官方化的解释

比较官方的解释是：

动态规划（Dynamic Programming）：简称 DP，是一种求解 多阶段决策过程最优化问题 的方法。在动态规划中，通过把原问题分解为相对简单的子问题，先求解子问题，再由子问题的解而得到原问题的解。

动态规划最早由理查德 · 贝尔曼于 1957 年在其著作动态规划（Dynamic Programming）一书中提出。注意这里的 Programming 并不是编程的意思，而是指一种表格处理方法，即将每一步计算的结果存储在表格中，供随后的计算查询使用。

2.2 思想

如果你理解了上面引入的案例，那么我就可以说说动态规划的思想了。主要是三点：

  1. 最优子结构
  2. 无后效性
  3. 子问题重叠

什么是最优子结构，它指的是一个问题的最优解包含其子问题的最优解，如何理解，以上面的案例来说，我们将 r=5r=5r=5 的解称为一个复杂的问题，那么 r=4r=4r=4 的解就是复杂问题的一个子问题。

而 r=5r=5r=5 的最优解是 7→3→8→7→57\rightarrow 3\rightarrow 8\rightarrow 7\rightarrow 57→3→8→7→5，r=4r=4r=4 的最优解是 7→3→8→77\rightarrow 3\rightarrow 8\rightarrow 77→3→8→7，是不是复杂问题的最优解包含了其子问题的最优解。

那么什么是无后效性呢？我的理解是，当一个子问题的解被确定了之后，那么求解后续问题时，将不会被修改。

最后，子问题重叠，前面两点应该是建立在这一点的基础之上的，而这也是动态规划优化的地方，即如果有大量的重叠子问题，那么我们可以使用额外的空间（如数组）将子问题的解记录下来，后续求解时就不用再求了。

当然你可能什么也看不懂，但其实做过了一些动态规划的题目之后，我对动态规划的理解就变成了，对于某一类问题（满足上面三个条件的问题），通过画一维、二维甚至三维的表格，按照题目的要求填充这个表格，如果填表的过程中有一些规律，那么还可以进行额外的优化，这个表格其实就是大家常说的 dp 数组（我最初当接触这个概念还感觉多么高大上，其实也就是一个数组），也可以看出，动态规划其实是一种用空间换时间的算法。

那么备忘录和 dp 数组的区别在哪？

那么如何选择备忘录或者 dp 数组呢？

备忘录算法采用 自然的递归 方式编写，也就是说，你能用好递归，那么就可以用好备忘录算法，当然可能会因为递归深度过大而导致栈溢出问题，实际很少见，要么就是你编写的代码递归终止不当。

而 dp 数组的方式，在动态规划进行进一步优化，比如使用四边形不等式优化动态规划时，你必须使用 dp 数组的方式画出表格分析。

3. 从递归到 dp

接下来，我讲一讲我入门动态规划的一种方式，就是将递归改写为动态规划，这在如背包问题中对于你的理解是很有帮助的。

方法来源

这里我先给出方法论，后面辅以几个例题说明。

3.1 方法论

  1. 首先拿到一个问题，用 自然智慧 尝试写出一个暴力递归（当然这或许也很难），并且明确确实存在解的重复调用。
  2. 找出哪些参数的变化会影响解！对每一个参数列出变化范围。
  3. 对于这些变化的参数间的所有的组合数量，决定了 dp 表的大小。
  4. 此时可以非常简单的在你写出的递归方法中加入 缓存 来记录子问题的解。
  5. 如果想要继续优化，那么根据可变参数的大小，规定严格的表（dp 表），分析位置的依赖关系，逐步填满整个表，返回最终解。
  6. 对于在填 dp 表的过程中，如果还有枚举行为，那么可能会有进一步优化

大致就是：

递归简单来说就是尝试，它将原问题转换为规模更小的同类的子问题，存在明确的终止递归的条件（base case），也存在当得到某个子问题的解之后的决策过程，如果在求解过程中不记录解，就是递归，而如果使用缓存记录问题的解，就是备忘录算法，如果使用严格的 dp 表结构记录问题的解，就是动态规划。

3.2 0-1 背包问题

下面我以 0-1 背包问题说明一下该方法。

0-1 背包问题：有 nnn 件物品和有一个最多能装重量为 WWW 的背包。第 iii 件物品的重量为 weight[i]weight[i]weight[i]，价值为 value[i]value[i]value[i]，每件物品有且只有 111 件。请问在总重量不超过背包载重上限的情况下，能装入背包的最大价值是多少？

在这个问题中，每个物品只有两种状态，也就是装不装入背包，这对应二进制中的 000 和 111，这类问题被称为 0-1 背包问题。

我们先尝试使用递归来求解：

  1. 问题是求能装入背包的最大价值，我们将其记为 maxmaxmax。
  2. 我们这样想，对于这 nnn 件物品，我们对每一件物品都做一个决策，即要不要装入背包，我们选择将该物品装入背包的情况下所得到的 maxmaxmax 与不装入背包的情况下所得到的 maxmaxmax 比较，较大值作为解。
  3. 那么对于第 iii 个物品不放入背包，那么它所获得的最大价值是第 i+1i+1i+1 个物品做决策所能获得的最大价值，而如果第 iii 个物品放入背包，那么它所获得的最大价值是第 i+1i+1i+1 个物品做决策所能获得的最大价值，再加上第 iii 个物品的价值总和。
