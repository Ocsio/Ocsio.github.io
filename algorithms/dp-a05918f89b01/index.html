动态规划(三)—背包 DP

背包 DP

在第一篇文章中，我以 0-1 背包为例，说明了如何将递归改写为动态规划，那么这一篇文章主要是更详细的讲一下背包问题相关的动态规划。

首先，背包问题是经典又特殊的一类 线性 问题，或者你也可以称其为一类模型。

其描述为：给定一组物品，每种物品都有自己的重量、价格以及数量。再给定一个最多能装重量为 WWW 的背包。现在选择将一些物品放入背包中，请问在总重量不超过背包容量的情况下，能装入背包的最大价值总和是多少？

根据物品限制条件的不同，背包问题可细分为：0-1 背包、完全背包、多重背包、分组背包，以及混合背包等，我会一一进行说明。

1. 0-1 背包

0-1 背包问题：有 nnn 件物品和有一个最多能装重量为 WWW 的背包。第 iii 件物品的重量为 weight[i]weight[i]weight[i]，价值为 value[i]value[i]value[i]，每件物品有且只有 111 件。请问在总重量不超过背包载重上限的情况下，能装入背包的最大价值是多少？

由于每个物品只有两种状态（取与不取），对应二进制中的 000 和 111，这类问题被称为 0-1 背包问题。

0-1 背包问题的特点：每种物品有且仅有 111 件，可以选择放入背包，也可以选择不放入背包。

1.1 dp 数组

首先，背包 DP 也是一种线性 DP 类问题，所以一般还是以物品的序号，当前背包的容量进行阶段划分。

定义状态 dp[i][w]dp[i][w]dp[i][w] 为前 iii 件物品放入一个容量为 www 的背包中，可以获得的最大价值。

接下来考虑状态转移：

考虑第 i−1i-1i−1 件物品（前 iii 件物品中的最后一件）的放入策略（放或不放）则转换为一个最优子问题。

  • 第 i−1i-1i−1 件物品不放入背包，则考虑前 i−1i-1i−1 件物品放入容量为 www 的背包可以获得的最大价值，即 dp[i−1][w]dp[i-1][w]dp[i−1][w]
  • 第 i−1i-1i−1 件物品放入背包，则考虑前 i−1i-1i−1 件物品放入容量为 w−weight[i−1]w-weight[i-1]w−weight[i−1] 的背包可以获得的最大价值，加上第 i−1i-1i−1 件物品的价值，即 dp[i−1][w−weight[i−1]]+value[i−1]dp[i-1][w-weight[i-1]]+value[i-1]dp[i−1][w−weight[i−1]]+value[i−1]，当然，当背包的容量不足以放入第 i−1i-1i−1 件物品时不能放入背包。

所以状态转移方程：
